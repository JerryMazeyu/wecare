hi, 小杨：
   1. 大概的想法是什么？
   我们的项目是一个web应用，简单的说，就是一个网页的形式。我想，现在很多人都会在工作中很忙而忽略很多对于自己和他人的关心，我想通过一款应用的方式，让每个人可以自己定制对自己或者他人的关心。举个例子，很多时候我们忘记看天气预报，而忘记带伞或者穿的不合适，而我们的应用就可以让用户自己定制，比如我想定制在第二天下雨的前一天的九点半给自己发一条短信：记得带伞哦。或者在温度高于30度的时候，给领导发一封邮件：今天很热，不要中暑烧坏了脑子。当然了，这个定制的服务也不见得是需要一定和天气有关的，但是我们可以一开始主要做天气的。
   2. 如果我们有未来，应该怎么去发展流量呢？
   真正的有流量的东西一定是让用户之间自己交互的，如果按照上面的话，我们没有办法创造用户之间的交流，也就不会有流量。所以我们的扩展，就是从一个单纯的服务，扩展到一个论坛，可能用这个应用需要注册账号，然后会方便用户之间交流。但我还没想好怎么让论坛活跃起来，这是后话了。
   3. 实现最初的应用需要做什么呢？
   （1）. 我们要搭建一个网站，这个框架用Django来搭载，这是一个基于python的前端框架，我们正好可以顺便研究研究python。
   （2）. Django搭建应用是要配一个数据库的，默认的数据库是mysql，我想也可以用nosql类型的数据库，比如mongodb这种，但是我一直没有成功，之前也一直卡在这里。数据库这一块我一直不太懂，希望你能多多指教。
   （3）. 做网站肯定不能在本机上面运行的，不然的话本机关机服务就关闭了，所以我们最后写好的代码要在虚拟机上跑，我们必须掌握一些虚拟机的基本知识才行。
   （4）. 既然我们要根据天气的变化来提醒用户，那么就会有两块内容需要掌握，一个是怎么在网站里获取天气，一个是怎么通知用户。获取天气的话，应该是写一个脚本，爬天气网的数据来获得，同时在写一个定时脚本，放在虚拟机里面跑起来；而通知用户的话，我想暂时有邮箱、短信两种方式；其中邮箱并不难，我好像已经写出来了（但我忘记放在哪里了。。。），短信的话要添加一个运行商的接口，应该还得花钱，但是问题不大，很便宜。
   （5）. 我们为了未来的用户，还要添加一个论坛的接口，简单的说，就是要在代码里面把论坛功能给分离出来，希望看到的结果是后面添加了论坛也不用重新修改代码，这个不是必要的，但我希望我们可以学会。
   （6）. 在做完这一系列工作后，希望我们的网站是稳健的，可以支持较多用户的同时使用的，这除了升级服务器之外，还需要高可用、高并发、多线程技术，我希望后面我们可以一起学习、使用。
   4. 需要注意哪些细节呢？
   （1）. 首先我们要找到我们合作的方式，寒假里面不可能面谈，这样的开发效率也并不高，所以我想我们把代码都放在github上面，这样便于合作开发，讲不定还有外来大神给我们一点援助呢。
   （2）. 我们的代码应该是简练、有扩展性、规范的，这样拿出去也不至于让别人笑话，所以我希望我们的代码格式符合pep8格式（这是一种）......
   5. 我们的日程是什么？
    首先，我们需要学习相关的知识，包括django、github、html、mysql、爬虫、虚拟机、分布式的相关知识。我们的参考文献主要是视频网站教程（b站）、相关网页（廖雪峰）、官网文档（***）、相关git库。
    其次，先从简单的完成，那么我们的顺序是：天气爬虫<发送邮件<定时器<<django框架=数据库<虚拟机安装部署<前端编写<高并发
